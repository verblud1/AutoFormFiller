# Работа с Git в AutoFormFiller

## Проблема с unstaged changes

Часто возникает проблема при выполнении `git pull`, когда Git сообщает о наличии несохраненных изменений (unstaged changes). Это происходит, когда вы вносите изменения в файлы, но не коммитите их.

## Решение проблемы

### Вариант 1: Сохранение изменений перед pull (рекомендуется)

```bash
# Проверьте статус репозитория
git status

# Добавьте измененные файлы
git add .

# Сделайте коммит
git commit -m "Описание ваших изменений"

# Выполните pull
git pull
```

### Вариант 2: Использование stash

```bash
# Сохраните изменения временно
git stash

# Выполните pull
git pull

# Восстановите изменения
git stash pop
```

### Вариант 3: Игнорирование файлов

Если у вас есть файлы, которые не должны быть в репозитории, добавьте их в `.gitignore`:

```
# Автосохранения
*autosave*

# Завершенные задания
files for fill queue/completed/*

# Временные файлы
*.tmp
*.temp
```

## Рекомендации по работе

1. **Регулярно коммитьте изменения** - не держите много времени несохраненные изменения
2. **Используйте осмысленные комментарии** к коммитам
3. **Проверяйте статус репозитория** перед выполнением операций:
   ```bash
   git status
   ```
4. **Используйте stash при необходимости** временно сохранить изменения
5. **Обновляйтесь перед началом работы**:
   ```bash
   git pull
   ```

## Частые ошибки и их решения

### Ошибка: "cannot pull with rebase: You have unstaged changes."

**Решение:** 
- Закоммитьте изменения или используйте `git stash` как описано выше

### Ошибка: "conflict during merge"

**Решение:**
- Разрешите конфликты вручную
- Добавьте файлы и закоммитьте
- Продолжите процесс merge

## Полезные команды

```bash
# Проверить, что изменено
git diff

# Посмотреть историю коммитов
git log --oneline

# Посмотреть сохраненные stash
git stash list

# Сбросить изменения в файле
git checkout -- имя_файла

# Сбросить все несохраненные изменения
git reset --hard HEAD
```

> ⚠️ ВНИМАНИЕ: `git reset --hard HEAD` удалит все несохраненные изменения! Используйте с осторожностью.
